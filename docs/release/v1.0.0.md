# git-cleaner v1.0.0

`git-cleaner` の最初の安定版リリースです。ベースブランチ（`develop` / `main` 等）にマージ済みのローカルブランチを安全に検出し、対話的に削除する CLI ツールです。

## 主な機能

- **マージ済みブランチの検出と対話削除**: ベースブランチに（通常マージで）取り込み済みのローカルブランチを抽出し、1 件ずつ確認しながら `git branch -d` で削除する。
- **Git config ネイティブな設定**: 独自の設定ファイルを持たず、Git 標準 config の `[cleaner]` セクションに `targets` / `protect` を統合。グローバル（`~/.gitconfig`）とローカル（`.git/config`）の値を結合（union）して評価する。`git-cleaner init` でテンプレートを冪等に追記できる。
- **保護ブランチ**: `cleaner.protect` で指定したブランチを削除対象から除外。カレントブランチは設定の有無にかかわらず常に保護される。
- **dry-run モード**: `-d` / `--dry-run` で実削除を行わず、削除対象の一覧と詳細のみを表示する。
- **ターゲットの一時上書き**: `-t` / `--target` で `cleaner.targets` をその場だけ上書きできる。
- **件数制限**: `-l` / `--limit` で削除対象を先頭 N 件に絞る（dry-run・対話削除の両方に適用）。
- **マージ情報の表示**: 各候補にマージ先・マージ日時＋マージコミット短縮ハッシュ・最終コミット日時（相対時間）＋コミットメッセージ 1 行目・リモートブランチの生存状態を表示する。
- **部分マージの警告**: 一度マージされた後にターゲット未取り込みのコミットを追加したブランチは `⚠` 付きで警告表示する。`git branch -d` の安全機構が削除を拒否するため、未マージ作業を誤って失わない。
- **未 push コミットの警告と force 削除**: マージ済みでも `origin/<同名>` へ未 push のローカルコミットがあるブランチは `⚠ 未 push` 警告を表示し、プロンプトを `(y/N/skip/force)` に切り替える。`force` / `f` で `git branch -D` を実行する。

## バグ修正

- 初回リリースのため該当なし。

## 内部改善

- 読み取り系（ブランチ列挙・マージ判定・履歴走査）は `git2`、変更系（`fetch --prune` / `branch -d` / `branch -D`）は `git` バイナリへのシェルアウトというハイブリッド構成を採用し、ユーザー既存の認証情報をそのまま再利用する。
- 対話・I/O ロジックは I/O を注入する設計とし、TTY なしでもプロンプトをテストできる。
- ユニットテスト（inline `#[cfg(test)]`）と統合テスト（`tempfile` + `git` CLI + `assert_cmd`）の両方を整備。
- GitHub Actions による CI（`check` / `test` / `fmt` / `clippy` を `-Dwarnings` で実行）とクロスプラットフォームのリリースビルドを整備。

## 対応プラットフォーム

- Linux / macOS / Windows（`git` バイナリと git2 が動作する環境）

## 既知の制限

- 検出対象は**通常マージ**（マージコミットが残る形式）のみ。Squash / Rebase マージ（コミットハッシュが変わる形式）は対象外。
- `git fetch --prune` と `git branch -d` / `-D` は `git` バイナリへシェルアウトする（リモート認証はユーザーの `git` 設定をそのまま利用）。
