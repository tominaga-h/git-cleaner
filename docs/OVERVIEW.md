# 要件定義書: git-cleaner (CLI Tool)

## 1. 概要
本ツールは、ローカル環境に溜まった作業ブランチや一時データを安全かつ効率的にクリーンアップするためのCLIツールである。
フェーズ1（MVP）では、Gitの履歴を参照し、指定されたベースブランチ（`develop`, `main` 等）に対してすでにマージ済みのローカルブランチを検出し、ユーザー確認のもと安全に削除する「ブランチ掃除機能」を提供する。

## 2. 技術スタック
* **開発言語**: Rust
* **主要クレート（想定）**:
  * `clap`: CLI引数・サブコマンドのパース（デフォルトサブコマンド機能の利用）
  * `git2` または `gix`: Gitリポジトリ操作、Configの読み書き
  * `chrono`: 日時情報のフォーマット表示

## 3. コマンド・引数仕様
将来的なクリーンアップ対象（StashやUntrackedファイル）の拡張を見据え、サブコマンド構造を採用する。ただし、利便性のために引数なしで起動した場合はデフォルトでブランチ削除処理（`branch` サブコマンド相当）が実行されるものとする。

### 3.1. 実行コマンド
```bash
# デフォルト（ブランチ削除処理が即座に起動）
git-cleaner

# 明示的なサブコマンド指定（上記と同じ挙動）
git-cleaner branch
```

### 3.2. オプション（`branch` サブコマンド共通）
* `-d`, `--dry-run`
  * 実際の削除処理は行わず、削除対象となるブランチの一覧と詳細情報の表示のみを行って終了する。
* `-t <BRANCH>`, `--target <BRANCH>`
  * Git Configに設定されたマージ先ターゲットを一時的にオーバーライドし、指定したブランチにマージ済みのものを抽出する。

### 3.3. 初期化コマンド
```bash
git-cleaner init
```
* **挙動**:
  1. グローバル設定ファイル（`~/.gitconfig`）を確認し、`[cleaner]` セクションが存在しなければ、グローバル用のテンプレート設定（コメント付き）を追記する。
  2. ローカル設定ファイル（カレントディレクトリの `.git/config`）に、プロジェクト固有用の `[cleaner]` セクションのテンプレート設定（コメント付き）を追記する。

---

## 4. 設定仕様（Git Config 統合）
独自の設定ファイルは作成せず、Git標準の設定システムに完全統合する。

### 4.1. 設定項目
* `cleaner.targets`: マージ先として判定するベースブランチ（カンマ区切り）。
* `cleaner.protect`: 削除対象から絶対に除外する保護ブランチ（カンマ区切り）。

### 4.2. グローバル設定（`~/.gitconfig`）の初期テンプレート
```ini
[cleaner]
	# PC全体で共通の保護ブランチ（デフォルト設定）
	protect = main,master
```

### 4.3. ローカル設定（`.git/config`）の初期テンプレート
```ini
[cleaner]
	# このプロジェクトにおけるマージ先ベースブランチ（カンマ区切り）
	# targets = develop

	# このプロジェクト固有で追加したい保護ブランチ（例: staging）
	# protect = staging
```

### 4.4. 設定の評価ロジック
* `cleaner.targets` および `cleaner.protect` は、グローバル設定とローカル設定の両方の値を**「結合（マージ）」**して処理を行う。
* カレントブランチ（現在チェックアウトしているブランチ）は、設定の有無に関わらず自動的に保護対象（削除不可）とする。

---

## 5. 処理フロー（フェーズ1: ブランチ削除）
引数なし、または `branch` サブコマンドが実行された際の内部プロセス。

1. **リモート情報の最新化**: 裏側で `git fetch --prune` を自動実行し、リモートの最新状態をローカルに同期する。
2. **設定の読み込み**: `git config` からターゲットブランチと保護ブランチのリストを取得し、結合する。
3. **対象ブランチの抽出**:
   * 通常マージ（マージコミットが残る形式）で、ターゲットブランチ（`cleaner.targets`）にマージ済みのローカルブランチを抽出。
   * 抽出リストから、保護ブランチ（`cleaner.protect`）およびカレントブランチを除外。
4. **対話型確認・削除**: 抽出されたブランチを1件ずつ画面に提示し、ユーザーの承認（y/N）をもとに `git branch -d` を実行する。

---

## 6. UI/UX 仕様 (プロンプト形式)
対象ブランチを1件ずつ画面に表示し、ユーザーが判断しやすい情報を付与して対話的に処理する。

### 6.1. 表示情報
* ブランチ名
* マージ先ターゲットブランチ名
* 最終コミット日時（相対時間も併記すると望ましい）
* リモートブランチの生存状態（`git fetch` の結果、リモートに対応するブランチが残っているか否か）

### 6.2. インタラクションイメージ
```text
[1/2] ブランチ 'feature/login-ui' は 'develop' にマージ済みです。
  - 最終コミット: 2026-05-25 14:30 (2日前)
  - リモート状態: 削除済み (GitHub等で削除されています)
? このローカルブランチを削除しますか？ (y/N/skip) > y
-> 削除しました: feature/login-ui

[2/2] ブランチ 'feature/api-fix' は 'develop' にマージ済みです。
  - 最終コミット: 2026-05-31 09:15 (今日)
  - リモート状態: 存在 (リモートにもまだ残っています)
? このローカルブランチを削除しますか？ (y/N/skip) > n
-> スキップしました。
```

## 7. 将来的な拡張性（フェーズ2以降）
以下の機能を別サブコマンドとして拡張できるよう、拡張性を担保した設計とする。

* **`git-cleaner stash`**:
  * 溜まったスタッシュ（`git stash`）の内容や日時を1つずつ提示し、対話的に破棄（`drop`）する機能。
* **`git-cleaner file`**:
  * `.gitignore` に指定されていない、ワークツリー上の未追跡ファイル（Untracked files）を対話的に掃除する機能。
* **`git-cleaner tag`**:
  * リモートで既に削除されている、古いローカルタグをクリーンアップする機能。
* **Squash / Rebase Merge への対応**:
  * コミットハッシュ値が変わるマージ形式のブランチに対しても、差分比較等を用いて検知する機能の検討。
